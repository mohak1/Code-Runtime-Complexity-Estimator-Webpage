Welcome to Code Runtime Complexity Estimator

Simply put, this website lets you estimate the runtime complexity of your code by tracking its execution time.
All you have to do is take your code and some input cases, fill them in the respective textareas and hit the "Submit" button.

Now, it all sounds quite straightforward but there are some things to keep in mind before you proceed:

How does it work?

The code and the input cases are sent to an online compiler for execution and their runtimes are noted.
These runtimes are then used as data points and are fitted to the runtime complexity equations (Logarithmic, Linear, Exponential, etc).
The equation that fits the best to these data points is declared as the runtime complexity for the code.

What could go wrong?

Since no analytical technique is used here for determining the runtime complexity of the code, it is important to realise
the importance of the inputs that are to be provided.
This means that if all the inputs are worst-case scenarios with increasing but equally spaced test cases, then it can be expected
to obtain the worst case time complexity of the code.
One of the main reasons of getting an incorrect estimate could be the use of insufficient inputs or using inputs that do not represent
the worse-case runtime. It is also important for the inputs to be equally spaced so that the change in runtime can be correctly determined.

Obtained runtime estimates: Best, Average or Worst-case?

Just as with every data-driven technique, this one also majorly depends on the inputs it gets to work with.
If all the inputs provided are worst-case scenarions, then with enough number of well spaced inputs you should obtain the worst-case estimate.
However, if the inputs are provided at random, with a mix of all runtime scenarios, then the estimated runtime complexity would most likely be average-case.

Bottom Line:

This method of estimating runtime complexity is neither foolproof nor scientific.
At best, it is a data driven way of achieving the same goal.
The results you obtain from this website very much depend on the inputs you provide for the code.
With enough number of well spread worst-case inputs, you should be able to obtain an accurate Big-O complexity estimate of your code.
